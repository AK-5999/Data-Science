| Step                                       | Gradient Boosting (GBM)                                                      | XGBoost (What Changes?)                                                                        | Benefit over GBM                                                                    |   |                                                  |
| ------------------------------------------ | ---------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- | - | ------------------------------------------------ |
| **1. Initial Prediction**                  | Regression: Mean of target values  (;F_0=\text{mean}(y))                     | Same idea. Starts with a base prediction that minimizes the loss.                              | No major difference.                                                                |   |                                                  |
| **2. Compute Error Signal**                | For MSE: Residuals  (;r_i=y_i-\hat y_i)  (which are also negative gradients) | Computes **Gradient** ((g_i)) and **Hessian** ((h_i)) for every sample.                        | Knows both **direction** and **curvature** of the loss. More accurate optimization. |   |                                                  |
| **3. What is Passed to Tree?**             | Residuals / Negative Gradients                                               | Gradient + Hessian pairs ((g_i,h_i))                                                           | Tree has richer information about the loss surface.                                 |   |                                                  |
| **4. Build Tree / Find Splits**            | Split chosen using reduction in impurity (typically MSE reduction).          | Split chosen using **Gain Formula**:  (\text{Gain}=\cdots-\gamma)                              | More mathematically optimal splits.                                                 |   |                                                  |
| **5. Where is Gamma ((\gamma)) Used?**     | Not present.                                                                 | Used during **split evaluation**. Every split must produce enough gain to overcome (\gamma).   | Prevents unnecessary leaves and controls tree complexity.                           |   |                                                  |
| **6. Split Acceptance Decision**           | If impurity decreases, split is usually accepted.                            | Split accepted only if Gain > 0 after gamma penalty.                                           | Better generalization, less overfitting.                                            |   |                                                  |
| **7. Leaf Formation**                      | Samples with similar residuals are grouped into leaves.                      | Samples with similar gradient-hessian statistics are grouped into leaves.                      | More informed grouping.                                                             |   |                                                  |
| **8. Leaf Value Calculation**              | Aggregator approach. For MSE:  (\gamma_j=\text{mean(residuals)})             | Optimization formula:  [w^*=-\frac{\sum g_i}{\sum h_i+\lambda}]                                | More optimal leaf corrections.                                                      |   |                                                  |
| **9. Where is Lambda ((\lambda)) Used?**   | Not present.                                                                 | Used in leaf weight formula denominator.                                                       | Shrinks leaf outputs and reduces overfitting.                                       |   |                                                  |
| **10. L1 Regularization (Alpha (\alpha))** | Not available natively.                                                      | Added in regularized objective.                                                                | Can push weak leaf weights toward zero, creating simpler models.                    |   |                                                  |
| **11. Objective Function**                 | Optimizes loss only.  [\text{Obj}=Loss]                                      | Optimizes regularized objective.  [\text{Obj}=Loss+\Omega(Tree)]                               | Controls complexity while minimizing loss.                                          |   |                                                  |
| **12. Regularization Term**                | No explicit tree regularization.                                             | [\Omega(Tree)=\gamma T+\frac{\lambda}{2}\sum w_j^2+\alpha\sum                                  | w_j                                                                                 | ] | Penalizes too many leaves and large leaf values. |
| **13. Optimization Method**                | First-order optimization (Gradient only).                                    | Second-order optimization (Gradient + Hessian).                                                | Faster convergence and better local approximation.                                  |   |                                                  |
| **14. Taylor Expansion**                   | Not explicitly used.                                                         | Uses **2nd-order Taylor Expansion** of the objective:  [L(y,\hat y+w)\approx L+gw+\frac12hw^2] | Enables closed-form leaf optimization and gain calculation.                         |   |                                                  |
| **15. Prediction Update**                  | [\hat y_{new}=\hat y_{old}+\eta h_m(x)]                                      | [\hat y_{new}=\hat y_{old}+\eta w_j]                                                           | Same boosting framework, but better leaf values.                                    |   |                                                  |
| **16. Missing Values**                     | Usually requires preprocessing/imputation.                                   | Learns default direction for missing values automatically.                                     | Less preprocessing effort.                                                          |   |                                                  |
| **17. Tree Growth Strategy**               | Standard CART growth.                                                        | Optimized growth using gain and pruning.                                                       | More efficient trees.                                                               |   |                                                  |
| **18. Pruning**                            | Limited or implementation-dependent.                                         | Post-pruning based on gain and gamma.                                                          | Removes weak branches automatically.                                                |   |                                                  |
| **19. Parallelization**                    | Sequential boosting; tree construction mostly standard.                      | Parallelized split finding.                                                                    | Faster training.                                                                    |   |                                                  |
| **20. Memory Optimization**                | Standard implementation.                                                     | Cache-aware and histogram optimizations.                                                       | Better scalability on large datasets.                                               |   |                                                  |
